# What is Docxa?

Docxa is an AI-powered documentation platform that treats documentation as a first-class engineering artifact. 

In many teams, documentation is either non-existent or perpetually stale. Docxa solves this by using AI to **extract truth** from code and **elicit intent** from stakeholders.

## Core Pillars

### 1. Reverse-Engineering from Code
Docxa doesn't just read code comments; it analyzes framework usage, architecture patterns, and repository structure to build a technical map of your project. This ensures your technical docs are based on what's actually in production.

### 2. Eliciting Stakeholder Intent
Technical docs need business context. Docxa uses a role-aware interview engine that asks the right questions to the right people. It normalized these answers into "business evidence."

### 3. Evidence-First Context
This is the "Secret Sauce." Docxa's Generation Planner ensures the AI is never working with incomplete information. It maps every section of a document (e.g., "Non-Functional Requirements") to specific evidence (e.g., "Latency goals from PM interview" + "Redis usage found in repo").

## The Goal: Documentation Intelligence
Docxa isn't just a generator; it's an intelligence system. It knows when your documentation is out of sync with your code and tells you exactly what steps to take to fix it.
